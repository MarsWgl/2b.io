Nginx location匹配规则

发表于 2018-11-20

location [=|~|~*|^~|@] /uri/ { … }

模式 含义
location = /uri = 表示精确匹配，只有完全匹配才生效
location ^~ /uri ^~ 开头对URL路径进行前缀匹配，并且在正则之前
location ~ pattern 开头表示区分大小写的正则匹配
location * pattern 开头表示不区分大小写的正则匹配
location /uri 不带任何修饰符，也表示前缀匹配，在正则匹配之后
location / 通常匹配，匹配所有请求
location @ 定义一个命名的 location，使用在内部定向时，例如 error_page, try_files

多个location匹配并不一定按照书写的顺序

  • 首先精确匹配 =
  • 其次前缀匹配 ^~
  • 其次是按文件中顺序的正则匹配（按照书写的顺序）
  • 然后匹配不带任何修饰的前缀匹配。
  • 最后是交给 / 通用匹配
  • 当有匹配成功时候，停止匹配，按当前匹配规则处理请求

CGI、FastCGI、PHP-CGI和PHP-FPM

发表于 2018-10-18

CGI和FastCGI

  • CGI和FastCGI都是一种通讯协议，是服务器(nginx,apache)与CGI程序之间的通讯桥梁

  • FastCGI是CGI的进化版，FastCGI比CGI性能更好

    CGI工作原理：每个请求到达之后，服务器会fork一个新的CGI解释器进程(php-cgi等)，CGI的进程在处理完请求后就自我销毁，新的请求又重新创建进程。每次创建进程就需要重新加载配置文件等，这样就会损耗性能，在并发下就不适合。

FastCGI工作流程:

  1. Web Server启动载入FastCGI进程管理器
  2. FastCGI进程管理器自身初始化，fork多个CGI解释器进程(php-cgi)并等待Web Server的连接
  3. 当客户端请求到达Web Server时，FastCGI进程管理器选择并连接到一个CGI解释器。程序完成后子进程关闭连接并等待下一个连接。
  4. 所以，FastCGI是常驻型的CGI，并不需要每一次的初始化，性能上比CGI要好上很多

PHP-CGI

php-cgi是PHP自带的FastCGI管理器

php-cgi变更php.ini配置需要重新启动才能生效，不能平滑启动

PHP-FPM

php-fpm也是一个FastCGI的管理器，早期是个补丁，现在PHP已经集成php-fpm

php-fpm优于php-cig的方面，改变配置文件后可以平滑的启动fastcgi进程
